Компонент `Підказка` відображає попередньо створену підказку ШІ, яку можна скопіювати до буфера обміну або відкрити безпосередньо в курсорі одним клацанням миші. Це ідеальний варіант для будь-якої сторінки з документацією, де ви хочете дати читачам готову підказку.

## Базове використання

<Prompt description="Підсумуйте цю кінцеву точку API">
Ви технічний письменник. На основі наведеної нижче документації кінцевої точки API напишіть короткий опис з одного абзацу, який пояснює, що робить кінцева точка, які вхідні дані вона вимагає і що вона повертає. Будьте лаконічними і використовуйте просту мову.
</Prompt>

```mdx
<Prompt description="Summarize this API endpoint">
Ви - технічний письменник. На основі наведеної нижче документації кінцевої точки API
напишіть короткий опис з одного абзацу, який пояснює, що робить кінцева точка, які
вхідні дані їй потрібні, і що вона повертає. Будьте стислими і використовуйте зрозумілу мову.
</Prompt>
```

## З інтеграцією курсору

Додайте `actions={["копіювати", "курсор"]}`, щоб також відобразити кнопку "Відкрити у курсорі". Це особливо корисно для підказок, які читачі використовуватимуть у своєму редакторі.

<Prompt description="Створення типізованого API-клієнта" icon="код" actions={["copy", "cursor"]}>




Ви досвідчений(-а) TypeScript розробник(-ця). За наведеною нижче специфікацією OpenAPI згенеруйте повністю типізований клієнт API з використанням нативної вибірки. Включайте:
* Базовий клієнтський клас з налаштованими заголовками baseUrl та auth
* Один метод на кінцеву точку з типізованими параметрами запиту та типами відповідей
* JSDoc коментує кожен метод
  Не використовуйте сторонні HTTP бібліотеки.
</Prompt>

```mdx
<Підказка
  description="Створити типізований клієнт API"
  icon="code"
  actions={["copy", "cursor"]}
>
Ви досвідчений розробник TypeScript. За наведеною нижче специфікацією OpenAPI
згенеруйте повністю типізований клієнт API з використанням нативної вибірки...
</Prompt>
```

## Реквізит

<ResponseField name="опис" type="рядок" required>
  Текст заголовка відображається над вмістом підказки. Підтримує уцінку.
</ResponseField>

<ResponseField name="діти" type="рядок" required>
  Текст підказки, який копіюється або надсилається курсору.
</ResponseField>

<ResponseField name="дії" type="масив">
  Які кнопки дій показувати. Опції: `"копіювати"`, `"курсор"`. За замовчуванням `["copy"]`.
</ResponseField>

<ResponseField name="іконка" type="рядок">
  Іконка, що відображається поряд з описом. Підтримує назви іконок Font Awesome, Lucide і Tabler, URL-адреси зображень або вбудовані SVG.
</ResponseField>

## Коли його використовувати

Компонент "Підказка" добре працює скрізь, де ви навчаєте читачів працювати зі штучним інтелектом:

* **Посібники зі швидкого старту** - дають читачам підказки для створення першої інтеграції
* **Довідкові сторінки API** - містять підказку для генерації клієнтського коду зі специфікації
* **Посібники з усунення несправностей** - додайте підказку з налагодження, яку читачі можуть запустити у власних журналах помилок
* **Концептуальні сторінки** - додайте підказку для подальшого вивчення теми за допомогою асистента зі штучним інтелектом
