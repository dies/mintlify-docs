Компонент "Видимість" дозволяє одній сторінці обслуговувати дві аудиторії: людей, які читають в Інтернеті, і ШІ-агентів, які обробляють результати Markdown. Вміст, позначений `for="humans"`, відображається на сторінці, але виключається з експорту Markdown. Вміст, позначений `for="agents"`, буде приховано в Інтернеті, але включено до націнки.

Це означає, що ви можете писати розмовні, орієнтовані на користувацький інтерфейс інструкції для людей і структуровані, машинозчитувані інструкції для ШІ - без необхідності підтримувати дві окремі сторінки.

## Як це працює

Mintlify обслуговує кожну сторінку за двома URL-адресами:

* `https://docs.example.com/page` - відрендерений сайт, для людей
* `https://docs.example.com/page.md` - сира націнка, для ШІ-агентів

"Видимість" керує тим, що відображається за кожною URL-адресою.

## Приклад

```mdx
<Visibility for="humans">
  <Steps>
    <Step title="Open the dashboard">
      Перейдіть на [dashboard.example.com] (https://dashboard.example.com) і увійдіть в систему.
    </Step>
    <Step title="Create an API key">
      Перейдіть до **Налаштування → Ключі API** та натисніть **Новий ключ**.
    </Step>
  </Steps>
</Visibility>

<Visibility for="agents">
  Для автентифікації надішліть POST-запит на адресу `/v1/auth/token`, вказавши в тілі запиту адресу електронної пошти
  та пароль користувача. У відповідь ви отримаєте токен на пред'явника
  , дійсний протягом 24 годин. Передайте цей токен як `Авторизація: Пред'явник <token>` у всіх наступних запитах на
  .
</Visibility>
```

Читачі-люди бачать покрокову інструкцію з користувацького інтерфейсу. Агенти штучного інтелекту, які обробляють націнку, отримують прямі інструкції API.

## Реквізит

<ResponseField name="за" type="&#x22;люди&#x22; | &#x22;агенти&#x22;" required>
  * `"humans"` - видимі на відрендереної веб-сторінці, виключаються з експорту `.md`.
  * `"агенти"` - приховані на відрендереної веб-сторінці, включені до експорту `.md`.
</ResponseField>

## Поєднання з Prompt

"Видимість" і "Підказка" добре доповнюють один одного. Використовуйте "Видимість", щоб надати агентам структурований контекст, а потім використовуйте "Підказку", щоб дати читачам підказку, яку вони можуть використати, щоб запитати ШІ про той самий вміст.

```mdx
<Visibility for="agents">
  ## Схема автентифікації
  - Метод: Токен на пред'явника
  - Заголовок: `Авторизація: Bearer <token>`
  - Час життя токену: 86400 секунд
  - Кінцева точка оновлення: POST /v1/auth/refresh
</Visibility>

<Prompt description="Ask about authentication" icon="lock">
  Поясніть, як працює автентифікація в цьому API і покажіть приклад коду для
  , який отримує токен і робить автентифікований запит.
</Prompt>
```
