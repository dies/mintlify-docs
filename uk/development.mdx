  * Node.js версії 19 або вище
  * Сховище документів з файлом \`docs.json

Виконайте ці кроки, щоб встановити та запустити Mintlify у вашій операційній системі.

<Steps>
<Step title="Встановіть Mintlify CLI">

```bash
npm i -g mint
```
</Step>

<Step title="Попередній перегляд локально">

Перейдіть до каталогу docs, де знаходиться ваш файл `docs.json`, і виконайте наступну команду:

```bash
mint dev
```

Локальний попередній перегляд вашої документації буде доступний на `http://localhost:3000`.

</Step>
</Steps>

## Користувацькі порти

За замовчуванням Mintlify використовує порт 3000. Ви можете налаштувати порт, на якому працює Mintlify, за допомогою прапорця `--port`. Наприклад, щоб запустити Mintlify на порту 3333, скористайтеся цією командою:

```bash
mint dev --port 3333
```

Якщо ви спробуєте запустити Mintlify на порту, який вже використовується, він буде використовувати наступний доступний порт:

```md
Порт 3000 вже використовується. Натомість спробуємо 3001.
```

## Mintlify версії

Зверніть увагу, що кожен випуск CLI пов'язаний з певною версією Mintlify. Якщо ваш локальний попередній перегляд не відповідає робочій версії, будь ласка, оновіть CLI:

```bash
npm оновлення mint
```

## Перевірка посилань

CLI може допомогти з перевіркою посилань у вашій документації. Щоб виявити непрацюючі посилання, скористайтеся наступною командою:

```bash
м'ятні биті посилання
```

Ми рекомендуємо використовувати розширення у вашому середовищі розробки для розпізнавання та форматування MDX. ## Усунення несправностей

<AccordionGroup>
  <Accordion title="Помилка: Не вдалося завантажити модуль &#x22;sharp&#x22; за допомогою середовища виконання darwin-arm64">

    Це може бути пов'язано із застарілою версією вузла. Спробуйте наступне:
    1. Видалити поточну версію CLI: `npm remove -g mint`.
    2. Оновлення до Node v19 або новішої версії.
    3. Перевстановіть CLI: `npm i -g mint`.
  </Accordion>

  <Accordion title="Проблема: Виникнення невідомої помилки">

    Рішення: Перейдіть у корінь вашого пристрою і видаліть папку `~/.mintlify`. Потім запустіть `mint dev` ще раз.
  </Accordion>
</AccordionGroup>

Цікаво, що змінилося в останній версії CLI? Ознайомтеся з [журналом змін у CLI] (https://www.npmjs.com/package/mintlify?activeTab=versions).
