Меню навігації - це список посилань на кожному сайті.

Скоріш за все, ви будете оновлювати `docs.json` щоразу, коли додаватимете нову сторінку. Сторінки не відображаються автоматично.

## Синтаксис навігації

Наш синтаксис навігації є рекурсивним, що означає, що ви можете створювати вкладені групи навігації. Вам не потрібно включати `.mdx` в назви сторінок.

<CodeGroup>

```json Regular Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Початок роботи",
          "сторінки": [ "quickstart"]
        }
      ]
    }
  ]
}
```

```json Nested Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Початок роботи",
          "сторінки": [
            "quickstart",
            {
              "group": "Вкладені довідкові сторінки",
              "pages": [ "nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## Папки

Просто розкладіть ваші MDX-файли по папках і оновіть шляхи в `docs.json`.

Наприклад, щоб створити сторінку за адресою `https://yoursite.com/your-folder/your-page`, вам потрібно створити папку з назвою `ваша-папка`, яка міститиме MDX-файл з назвою `ваша-сторінка.mdx`.

<Warning>

Ви не можете використовувати `api` для назви теки, якщо тільки ви не вкладаєте її в іншу теку. Mintlify використовує Next.js, який резервує папку верхнього рівня `api` для внутрішніх викликів сервера. Буде прийнятною назва теки, наприклад, `api-reference`.

</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Назва групи",
          "сторінки": [ "your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## Приховані сторінки

MDX-файли, не включені в `docs.json`, не відображатимуться на бічній панелі, але будуть доступні через рядок пошуку і за прямим посиланням на них.
