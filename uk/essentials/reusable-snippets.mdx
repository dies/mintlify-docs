Будь-який контент, який повторюється на різних сторінках, є хорошим кандидатом на створення сніппету. Створіть файл один раз у каталозі `snippets/`, імпортуйте його куди завгодно, і оновлення поширюватимуться всюди автоматично.

<Note>
  Файли всередині `сніппетів/` не відображаються як окремі сторінки. Вони з'являються тільки там, де ви їх імпортуєте.
</Note>

## Фрагменти вмісту

Найпростіший шаблон: запишіть MDX-вміст у файл фрагмента та імпортуйте його як компонент.

```mdx snippets/prerequisites.mdx
Перш ніж почати, переконайтеся, що у вас є
- Node.js 18 або новішої версії
- Ключ API з вашої [інформаційної панелі](https://dashboard.example.com)
```

Імпортуйте його на будь-яку сторінку, використовуючи кореневий шлях:

```mdx your-page.mdx
---
title: Встановлення SDK
---

import Prerequisites from '/snippets/prerequisites.mdx';

<Prerequisites />

## Встановлення
...
```

Ви також можете використовувати відносний імпорт, який дозволяє переходити до фрагмента у вашому редакторі за допомогою комбінації клавіш CMD+клацання миші:

```mdx
імпортувати Prerequisites з './snippets/prerequisites.mdx';
```

## Змінні фрагменти

Експортуйте іменовані константи з файлу фрагмента, щоб повторно використовувати такі значення, як назви продуктів, номери версій або URL-адреси.

```mdx snippets/vars.mdx
export const sdkVersion = '3.2.0';
export const baseUrl = 'https://api.example.com/v1';
```

Імпортуйте та використовуйте їх у потоці:

```mdx your-page.mdx
import { sdkVersion, baseUrl } from '/snippets/vars.mdx';

Поточна версія SDK - **{sdkVersion}**.

Всі запити надсилайте на `{baseUrl}`.
```

## Фрагменти компонентів

Для фрагментів, які мають змінюватися залежно від того, де вони використовуються, експортуйте компонент функції стрілки, який приймає пропси.

```mdx snippets/endpoint-note.mdx
export const EndpointNote = ({ метод, шлях }) => (
  <Note>
    Цей посібник описує кінцеву точку <code>{method} {path}</code>.
    Докладнішу інформацію можна знайти в <a href="/api-reference/introduction">довіднику API</a>. </Note> <Note>
  </Note>
);
```

* Використовуйте синтаксис стрілочної функції (`const Foo = () => ...`).
* Ключове слово `function` не підтримується у файлах фрагментів.
* Синтаксис MDX не компілюється всередині тіл стрілочних функцій. Використовуйте звичайні HTML-теги або скористайтеся експортом за замовчуванням.

Передавайте пропси під час рендерингу:

```mdx your-page.mdx
імпортуйте { EndpointNote } з '/snippets/endpoint-note.mdx';

<EndpointNote method="POST" path="/v1/users" />
```

## Імпорт сніппетів Імпорт сніппетів

Фрагменти можуть імпортувати інші фрагменти, що корисно для створення більших спільних розділів з менших фрагментів.

```mdx snippets/auth-intro.mdx
import AuthNote from '/snippets/auth-note.mdx';

## Автентифікація

Всі кінцеві точки потребують токен пред'явника.

<AuthNote />
```
